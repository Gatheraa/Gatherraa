# Architectural Decision Records (ADRs)

This directory contains records of significant architectural decisions made for the Gathera project.

## Index

- [ADR-001: Modular Contract Suite](./adr-001-modular-contract-suite.md) - Rationale for dividing the protocol into specialized contracts.
- [ADR-002: Soroban Storage Strategy](./adr-002-soroban-storage-strategy.md) - Decisions on use of Temporary, Persistent, and Instance storage.
- [ADR-003: Access Control Model](./adr-003-access-control-model.md) - Designing a unified role-based access control system.
- [ADR-004: Frontend State Persistence](./adr-004-frontend-state-persistence.md) - Handling form state and blockchain interactions.

## How to Create an ADR

1.  Use the `adr-XXX-title.md` naming convention.
2.  Follow the template:
    - **Status**: (Proposed, Accepted, Superceded)
    - **Context**: What is the problem we are solving?
    - **Decision**: What did we decide and why?
    - **Consequences**: What are the trade-offs and future implications?
